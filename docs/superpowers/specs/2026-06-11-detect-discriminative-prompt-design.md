# 设计：判别式 DETECT_PROMPT + Top-3 排序评测

- 日期：2026-06-11
- 状态：已批准，待实现
- 范围：仅 detect 阶段的 prompt 与配套评测；不动架构 / verify / 模型 / 并发

## 背景与问题

跑 `main.py` 测 1.jpg 与 2.jpg，pipeline 全程无报错但均未命中 Waldo：

| 图 | patch 数 | detect 判「有」 | 占比 | top-3 detect_conf | verify 结果 |
|----|---------|----------------|------|-------------------|------------|
| 1.jpg | 77 | 44 | 57% | 全 0.99 | 全否 |
| 2.jpg | 35 | 32 | 91% | 全 0.99 | 全否 |

**根因**：`prompts.py` 的 `DETECT_PROMPT` 要求「看到任意特征（红/白/条纹/帽子/眼镜）就 present=true」。在到处是红白色块的 Where's Waldo 图里，几乎每个 patch 都满足，于是返回 `present=true, confidence=0.99`。

`detect_node`（`agent/nodes/detect.py:98-99`）依赖**按 confidence 降序排序**后由 verify 取 top-3。当 90% 的 patch 都是 0.99，排序退化成随机，真 Waldo 的 patch 排不进前 3，verify 收到的候选里根本没有它 → 必然 miss。

**评测盲区**：现有 `tests/check_detect_prompt.py` 只用 `images_quicktests/`（全是含 Waldo 的正样本）测召回率，没有任何负样本，所以「召回 100%」漂亮、精确率塌到≈0 却没被发现。这次必须连评测口径一起修。

## 目标

让 detect 的置信度成为**有效的排序信号**：真 Waldo 的 patch 能排进 top-3，从而进入 verify。

**主指标：Top-3 命中率**（正样本置信度排进该图前 3 的图占比）。
辅指标：召回率（不能掉）、负样本误报率（要大幅降）、正负样本平均置信度 gap。

## 方案选择

已对比三方案，选**方案 1：判别式 prompt + 保留 `gpt-5.4-mini`**——最小改动、可量化，且与方案 3（换 `gpt-5.5`）共用同一套判别式 prompt，先验证 prompt 效果再决定是否升级，路径最干净。方案 1 失败时用评测数据客观决定是否升级。

## 组件 A：评测脚本 `tests/eval_detect_ranking.py`

### 数据来源

- 图片：`original-images/N.jpg`，N = 1..19。
- 真值：`original-images/bbox`，第 N 行 = `[x1, y1, x2, y2]`（左上+右下角）。
- **第 16 行为空**（16.jpg 无真值）→ 评测时跳过该图。

### 每张图的 patch 构造

- **正样本 ×1**：以真值框中心裁一个 **200×200** 窗口；窗口贴到图像边界时 clamp 回界内，保证仍包含完整真值框。
- **负样本 ×9**：随机裁 200×200 窗口，约束 = 与真值框**零重叠**（交集面积为 0）；落在图像内。
- 每图共 **10 个 patch**。
- **固定随机种子**（如 `SEED = 42`）：保证改 prompt 前后用同一批负样本，前后对比 apples-to-apples。

### 评分与指标

对 10 个 patch 各调一次 `vlm.detect()`，取 `confidence`。

- **Top-3 命中率（主）**：正样本 confidence 在该图 10 个 patch 中排名 ≤ 3 的图占比。
  - **平局悲观处理**：正样本与负样本同分时，正样本排在所有并列者**之后**。等价实现：正样本严格排名 = `1 + (置信度 > 正样本的负样本数) + (置信度 == 正样本的负样本数)`，命中条件 `严格排名 ≤ 3`。这样当前「全 0.99」破 prompt 会被如实判为接近随机基线（约 30%）。
- **召回率（辅）**：正样本 `present == true` 的图占比。
- **负样本误报率（辅）**：被判 `present == true` 的负样本数 / 负样本总数。
- **置信度 gap（辅）**：正样本平均 confidence − 负样本平均 confidence。

### 输出

- 每图一行：正样本 conf、正样本排名、是否命中 top-3、负样本 conf 列表摘要。
- 汇总：Top-3 命中率、召回率、负样本误报率、gap。
- `VLM_PROVIDER` / `VLM_MODEL` 顶部常量可调（默认对齐 detect 节点：`gpt4o` / `gpt-5.4-mini`）。
- 随机种子、负样本数 `N_NEG=9`、patch 尺寸 `PATCH_PX=200` 均为顶部常量。

### 调用量与限流

19 图（跳过 16 → 18 图）× 10 = 180 次调用。受 50 req/min 限制，复用 detect 节点的串行 + 指数退避策略即可（可直接调 `vlm.detect`，脚本层不必并发）。

## 组件 B：重写 `prompts.py` 的 `DETECT_PROMPT`

保持输出结构 `{"present": true/false, "confidence": 0.0-1.0}` 不变（`DetectResult` 解析、`DETECT_CONFIDENCE_THRESHOLD=0.15`、排序逻辑零改动）。

新 prompt 要点：

1. **判据收紧**：present=true 的核心条件 = 红白**横条纹**出现在**同一个人的躯干**上（Waldo 独有信号）。零散红色/白色、红白分布在不同物体上、纯红色无横条纹 → 不构成强信号。
2. **显式警告**：明确告诉模型这是密集人群，红、白、条纹在无关物体上到处都是，不要因为存在红白像素就判 true。
3. **置信度校准锚点**：
   - 0.8–1.0：单人躯干上清晰的红白横条纹（如同时有红白绒球帽 / 圆框眼镜则更确定）
   - 0.4–0.7：单个人物身上部分可见 / 被遮挡的条纹，像但不确定
   - 0.1–0.3：仅弱线索（红色上衣但无清晰白条纹，或图案在非人物上）
   - 0.0：任何位置都没有共址的红白横条纹
4. **保留召回**：真实但部分可见 / 被裁切的 Waldo 仍 present=true，但给较低非零分（>0.15 仍通过阈值）。目的是召回不掉、同时把置信度拉开，使排序恢复有效。

## TDD / 验证流程

1. 先写评测脚本，用**旧 prompt** 跑出基线（预期：Top-3 命中率 ≈ 随机基线 ~30%、负样本误报率高、gap ≈ 0）。
2. 应用**新 prompt**，重跑。
3. **判定规则（通过条件）**：Top-3 命中率显著高于基线 **且** 召回率不低于旧值。
4. 不达标 → 用评测数据决定升级到方案 3（detect 换 `gpt-5.5`，复用同一判别式 prompt）。

## 不在范围内（YAGNI）

- 不改 pipeline 架构、graph、verify 节点
- 不换 detect 模型（除非验证判定需要）
- 不改并发 `MAX_CONCURRENT`
- 不改 `images_quicktests/` 与旧的 `check_detect_prompt.py`（保留，新脚本独立）

## 旧 prompt 基线（实测，gpt-5.4-mini，SEED=42）

- Top-3 命中率：61.1%（11/18）
- 召回率：88.9%
- 负样本误报率：57.4%
- 置信度 gap：+0.311

观察：负样本误报率高达 57%，多张图负样本 conf 飙到 0.98–0.99 把正样本挤出 top-3（5/9/10/18 等）；17.jpg 正样本仅 0.06，召回直接落空。通过条件 = 新 prompt 的 Top-3 命中率显著高于 61.1% 且召回率不低于 88.9%。
